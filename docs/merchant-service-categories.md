# 6.1 商户服务分类建议

本工程已按 `6.1 商户服务分类建议` 建立商户服务分类契约。分类数据由服务端统一输出，商户端第 31 页从接口读取并可选择分类。

## 分类矩阵

| 大类 | 示例服务 | 备注 |
| --- | --- | --- |
| 医疗卫生 | 体检、问诊预约、陪诊协作、康复建议、附近医院服务 | 仅做预约与服务协调，不提供医疗诊断结论。 |
| 康养护理 | 护工护理、康复理疗、上门护理、慢病管理服务 | 需商户资质审核。 |
| 生活服务 | 家政、洗衣、保洁、维修、代购代办 | 可标准化套餐。 |
| 交通出行 | 接送站、包车、代驾、无障碍出行 | 需司机/车辆资质。 |
| 文旅体验 | 景区门票、讲解、活动课程、旅拍、非遗体验 | 与活动地图联动。 |
| 餐饮与本地美食 | 餐厅预订、营养餐、特色美食推荐 | 可结合老人饮食偏好。 |
| 殡葬服务 | 咨询、预约、方案服务、后续关怀 | 需谨慎合规，避免强推，页面表达要克制。 |

## 工程位置

- 分类契约：`server/lib/merchant-service-categories.js`
- 商户接口：`GET /api/merchant/service-categories`
- 图示裸路径：`GET /merchant/service-categories`
- 商户端页面：`/merchant/#31`
- 总参考接口：`GET /api/reference` 中的 `merchantServiceCategories`
- 自动验收脚本：`npm run test:merchant-categories`

## 当前实现边界

- 医疗卫生仅表达预约、协作和信息协调，不在页面或接口中生成诊断结论。
- 文旅体验分类已通过 `related.activitiesMap` 指向活动地图接口，后续活动课程和地图点可复用该分类。
- 殡葬服务保留合规提示，页面按钮只做分类选择，不做强推式入口。
