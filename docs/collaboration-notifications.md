# 8. 跨端协同与通知机制

本工程已按 `8. 跨端协同与通知机制` 建立协同规则契约，并补齐核心消息触发点。当前 MVP 使用站内消息完成跨端通知，短信、电话、微信订阅消息、App Push 保留集成接口。

## 协同规则

| 场景 | 触发端 | 接收端 | 通知/处理规则 |
| --- | --- | --- | --- |
| 用户下单 | 用户端 | 管理后台、人工向导端或商户端 | 生成订单；根据服务类型自动或人工派单。 |
| SOS 求助 | 用户端/设备 | 紧急联系人、管理后台、人工向导端 | 最高优先级；弹窗/短信/电话/站内消息多渠道通知。 |
| 设备异常 | 智能设备 | 用户端、家属、管理后台 | 异常入库，触发提醒，严重异常进入待处理列表。 |
| 人工向导接单 | 人工向导端 | 用户端、管理后台 | 用户看到订单状态变更和向导信息。 |
| 商户确认预约 | 商户端 | 用户端、管理后台 | 用户收到确认结果和服务方案。 |
| 服务完成 | 人工向导/商户端 | 用户端、管理后台 | 用户确认/评价，后台沉淀服务数据。 |
| 异常上报 | 人工向导/商户端 | 管理后台、用户端视情况 | 后台处理后决定是否通知用户或家属。 |

## 工程位置

- 协同规则契约：`server/lib/collaboration-notifications.js`
- 后台查询接口：`GET /api/admin/system/collaboration`
- 消息中心接口：`GET /api/messages?role=user|family|guide|merchant|admin`
- 异常处理接口：`GET /api/alerts`、`POST /api/alerts/{id}/handle`
- 自动验收脚本：`npm run test:collaboration`

## 已补齐的真实触发点

- 用户下单：创建订单后通知后台；后台派单后通知向导或商户。
- SOS 求助：通知后台、家属、向导，并写入高优先级异常。
- 设备异常：设备低电量、离线或异常状态会生成异常记录，并通知用户、家属、后台。
- 人工向导接单：通知用户和后台。
- 商户确认预约：报价/确认时通知用户和后台。
- 服务完成：向导/商户提交完成后通知用户确认评价，同时通知后台沉淀数据。
- 异常上报：向导和商户都可上报异常，后台接收，用户按规则收到提醒。
