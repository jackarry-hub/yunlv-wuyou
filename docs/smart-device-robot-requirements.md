# 4.6 智能设备与小云机器人需求

本工程已按 `4.6 智能设备与小云机器人需求` 建立服务端契约和用户端真实动作。

## 功能矩阵

| 功能点 | 详细需求 | 优先级 | 验收标准 | 工程实现 |
| --- | --- | --- | --- | --- |
| 今日健康概览 | 展示心率、血压、血氧、睡眠、步数等状态条。 | P0 | 可展示模拟/真实数据，异常状态颜色区分。 | `GET /api/devices/robot-requirements` 聚合健康指标，`GET /api/health/overview` 保留原健康概览。 |
| 设备状态 | 智能手环、小云机器人分别显示连接状态、电量、最后同步时间。 | P0 | 设备卡片独立展示，不混淆硬件。 | 手环和机器人按硬件角色独立返回，用户端分别渲染卡片。 |
| 设备联动 | 展示手环→平台→机器人→家人/紧急联系人的联动关系。 | P1 | 联动图或数据说明可见。 | 契约返回 `linkage.steps`，用户端展示联动链路。 |
| 小云机器人子页面 | 显示在线状态、电量、网络、语音音量、设备状态。 | P0 | 设备状态可视化展示。 | `robotStatus` 返回在线、电量、网络、音量和设备状态。 |
| 守护功能 | 摔倒检测、异常检测、离家/长时间未动提醒、生命体征检测、用药提醒、SOS 紧急呼叫。 | P0 | 各功能有开关或状态展示。 | `POST /api/devices/{id}/action` 更新守护开关，SOS 调用 `/api/alerts/sos`。 |
| 家人通话 | 展示家属联系人，支持语音/视频通话入口。 | P1 | 点击可调起通话或占位模拟。 | 通话动作写入设备 `lastCall`，并通知家属和用户消息。 |
| 寻求他人帮助 | 可联系附近求助、社区工作人员、人工向导，一键求助。 | P1 | 可生成帮助任务。 | `POST /api/devices/help-request` 生成后台服务请求，SOS 入口生成高优先级异常。 |

## API

- `GET /api/devices/robot-requirements` / `GET /devices/robot-requirements`：返回 4.6 需求契约、运行态、健康概览、设备状态、联动链路、机器人状态、守护功能、通话联系人和求助渠道。
- `POST /api/devices/{id}/action`：设备动作。支持守护开关、语音/视频通话、设备自检。
- `POST /api/devices/help-request` / `POST /devices/help-request`：生成智能设备帮助任务，写入后台服务请求。
- `POST /api/alerts/sos` / `POST /alerts/sos`：生成 SOS 异常，并通知后台、家属和向导。

## 验收脚本

```bash
npm run test:smart-device
```

该脚本会启动临时服务，验证 7 个需求点、P0/P1 数量、健康指标、独立设备卡、守护开关、家人通话、帮助任务、裸路径和 `GET /api/reference`。
