# 商户端功能总览

本工程已按图示商户端功能表建立可查询、可校验、可在商户工作台展示的工程契约。

## 功能矩阵

| 模块 | 功能需求 | 优先级 | 验收标准 |
| --- | --- | --- | --- |
| 商户入驻 | 机构信息、营业资质、服务类型、联系人、地址、审核状态 | P0 | 后台审核通过后才能上架服务。 |
| 服务管理 | 新增/编辑服务项目、图片、价格、服务说明、可预约时间 | P0 | 用户端可查看已上架服务。 |
| 订单/预约管理 | 接收平台订单或预约，查看客户需求、时间、地点、联系人 | P0 | 订单可确认、拒绝、改期。 |
| 报价确认 | 对复杂服务进行方案确认和报价，如医疗陪诊套餐、护理方案、殡葬服务方案 | P1 | 报价可回传用户端确认。 |
| 服务执行 | 安排服务人员、记录服务过程、上传完成凭证 | P0 | 服务完成后状态回传平台。 |
| 售后与评价 | 查看用户评价，处理投诉，跟进售后 | P1 | 商户能查看评价与反馈。 |
| 数据统计 | 订单数、成交额、好评率、服务类型占比 | P1 | 可按时间筛选。 |
| 结算管理 | 结算记录、待结算金额、已结算金额、发票信息 | P1 | 展示结算数据，首期可手动结算。 |

## 工程入口

- 商户端展示：`/merchant/#24`
- 商户功能接口：`GET /api/merchant/functions/overview`
- 总参考接口：`GET /api/reference` 中的 `merchantFunctionOverview`
- 状态流转：`POST /api/merchant/orders/{id}/quote|confirm|reject|reschedule|complete`
- 验收脚本：`npm run test:merchant-functions`

## 当前实现边界

- P0 已接入：商户入驻状态、服务管理、订单确认/拒绝/改期、服务完成回传。
- P1 已接入：报价回传、评价售后、统计和结算数据展示。
- 图片上传、支付结算、发票系统和真实客服通道仍按 MVP 边界保留接口与前端入口，后续替换为真实对象存储、支付和工单系统。
