# 4.4 活动地图需求

本工程已按 `4.4 活动地图需求` 建立活动地图接口与用户端真实联动。

## 需求矩阵

| 功能点 | 详细需求 | 优先级 | 验收标准 |
| --- | --- | --- | --- |
| 地图展示 | 基于弥勒区域展示活动位置点，支持地图缩放、定位、刷新。 | P0 | 可看到多个活动点和当前定位。 |
| 活动分类 | 全部、文化体验、康养健身、休闲娱乐、自然观光、学习讲座等。 | P0 | 分类筛选后地图点与列表同步变化。 |
| 活动卡片 | 地图点点击出现活动卡片，展示名称、时间、地点、距离、状态。 | P0 | 活动信息显示正确。 |
| 附近推荐 | 底部横滑展示附近活动推荐。 | P0 | 活动卡片可进入详情。 |
| 活动报名 | 活动详情页支持报名、取消报名、查看参与人数。 | P1 | 报名后生成记录并消息提醒。 |

## 工程接口

| 接口 | 说明 |
| --- | --- |
| `GET /api/activities/map-requirements` | 返回 4.4 需求契约、弥勒中心点、当前定位、分类、地图点、附近推荐和用户报名记录。 |
| `GET /api/activities/map?category=文化体验` | 返回地图点，可按分类筛选，地图点与列表用同一份数据。 |
| `GET /api/activities/{id}` | 返回活动详情、卡片字段、参与人数、剩余名额、当前用户报名状态。 |
| `POST /api/activities/{id}/join` | 活动报名，按报名人数更新活动人数并生成用户/后台消息。 |
| `POST /api/activities/{id}/cancel` | 取消报名，回退活动人数并生成用户/后台消息。 |

以上接口同时支持图示裸路径，例如 `/activities/map-requirements`。

## 数据闭环

- 活动数据来自 `data/mock-db.json` 的 `activities`，已覆盖弥勒区域 6 个活动点和 6 个分类。
- 报名数据写入 `activitySignups`，状态包括 `已报名`、`已取消`。
- 报名、取消报名会同步活动 `joined` 人数，并写入用户和后台消息。
- 用户端高德地图点位点击会先显示活动卡片，再进入活动详情/报名页。

## 验收脚本

```bash
npm run test:activity-map
npm run check
```

验收脚本会启动临时服务，验证需求契约、分类筛选、活动卡片字段、附近推荐、报名、取消报名、消息提醒和裸路径别名。
