## 已建立的自动检查

`scripts/visual-reference-mapping-test.js` 检查以下内容：

- 本地参考图数量必须为 209 张，且用户端 01-40、向导端 01-46、商户端 01-70、管理后台 01-53 编号连续。
- 三端正式页面路由必须全部存在于 `uniapp/pages.json`，对应 `.vue` 文件必须存在。
- 三端通用参考页模板必须带移动端安全区、卡片、按钮、参考图编号绑定等视觉基线。
- 三端所有参考页必须挂载正式参考页模板或已实现的核心闭环页；核心页覆盖用户首页/人工向导/活动地图/SOS/订单/消息/我的、向导接单/详情/服务/异常/收入、商户工作台/服务/订单/报价/完成/评价。
- 移动端源码和核心模板不得出现泛化状态反馈、空泛成功反馈或工程联调入口。
- 管理后台必须包含用户健康、向导审核、商户审核、服务评价、活动内容、订单派单、异常处理、数据闭环、系统验收等真实联动入口。

## 后续逐页精修规则

1. 每次按端和编号成批推进，例如用户端 08-15、向导端 01-10、商户端 24-30、管理后台 01-10。
2. 每页先对照参考图拆布局、文字、图标、颜色、间距和底部栏/侧边栏状态，再接入对应真实数据或后台接口。
3. 所有按钮必须落到真实动作：路由跳转、接口提交、状态流转、消息已读、派单、审核、上下架、异常处理或数据记录。
4. 每批完成后运行 `npm run test:visual-references`、`npm run test:admin-console` 和 `npm run check`。
