# 向导端功能总览

本工程已按用户提供的 `向导端功能总览` 图示建立工程契约，覆盖向导端 P0/P1 模块、页面入口、接口能力和运行状态。

## 模块矩阵

| 模块 | 功能需求 | 优先级 | 验收标准 |
| --- | --- | --- | --- |
| 登录与认证 | 手机号登录、身份资料、服务类型、实名/资质审核状态 | P0 | 通过审核后才能上线接单。 |
| 首页工作台 | 头像、姓名、在线状态、今日工作概览、今日收入、消息、扫一扫 | P0 | 显示接单数、服务中、已完成、今日收入。 |
| 上线接单 | 开启/关闭接单、设置服务区域、服务类型、接单提醒 | P0 | 开关后后台可看到状态。 |
| 接单大厅 | 推荐/附近/最新订单列表、按服务类型、距离、时间筛选 | P0 | 可查看待接订单并抢单/接单。 |
| 订单详情 | 客户信息、服务类型、时间地点、服务内容、注意事项、地图位置、预计收入 | P0 | 详情清晰，点击可接单。 |
| 服务中 | 开始服务、路线导航、联系客户、服务清单打勾、服务完成 | P0 | 订单状态从已接单→服务中→已完成流转。 |
| 异常上报 | 客户临时取消、客户失联、时间变更、突发情况、其他问题，支持文字和图片 | P0 | 上报后后台可看到异常。 |
| 我的订单 | 全部、服务中、已完成、已取消订单管理 | P0 | 订单列表与状态正确。 |
| 收入与评价 | 今日收入、预计收入、服务评价、结算记录 | P1 | 收入统计可按日/月查看。 |
| 帮助中心 | 服务规范、常见问题、联系客服 | P1 | 可查看规范文档。 |

## 工程位置

- 契约实现：`server/lib/guide-function-overview.js`
- 查询接口：`GET /api/guide/functions/overview`
- 图示裸路径：`GET /guide/functions/overview`
- 向导端展示入口：`/guide/#14`
- 总参考接口：`GET /api/reference` 中的 `guideFunctionOverview`
- 自动验收脚本：`npm run test:guide-functions`

## 当前实现边界

- 功能总览已接入真实模拟数据库，运行状态会读取向导认证、上线状态、待接单、任务、异常、收入和消息数据。
- `上线接单` 通过 `POST /api/guide/online` 修改状态后，功能总览会同步显示后台可见的在线状态。
- `接单大厅`、`订单详情`、`服务中`、`异常上报`、`我的订单` 均保留对应页面入口和 API 关联，首期按本地状态机完成演示闭环。
