# 云旅无忧 各端应用工程结构说明

> 版本：v1.0 | 日期：2026-06-14

---

## 一、整体架构

```
云旅无忧
├── server.js              ← 统一后端入口（Node.js HTTP Server）
├── server/lib/            ← 后端业务模块（26个模块）
├── apps/                  ← 四端前端应用源码
│   ├── shared/            ← 跨端共享层（API客户端 + 业务桥接）
│   ├── admin/             ← 管理后台
│   ├── guide/             ← 向导端
│   └── merchant/          ← 商户端
├── uniapp/                ← uni-app 跨端移动工程（小程序/App）
├── dist-preview/          ← 构建预览产物（GitHub Pages）
├── 云旅无忧UI界面参考图/    ← 四端UI实现代码（原始静态版本）
├── data/                  ← 种子数据
├── database/              ← 数据库脚本
├── deploy/                ← 部署配置
├── scripts/               ← 测试与构建脚本
└── docs/                  ← 项目文档
```

## 十、AI 管家服务

### 10.1 架构

```
用户端 → /api/ai/chat → server.js → DeepSeek API → AI回复
                                  ↘ (无Key时) → 本地模拟回复
```

### 10.2 配置

| 环境变量 | 说明 |
|----------|------|
| DEEPSEEK_API_KEY | DeepSeek API 密钥 |
| DEEPSEEK_MODEL | 模型名称（默认 deepseek-chat） |
| DEEPSEEK_API_BASE | API 基础地址 |

### 10.3 能力

- 文字对话（支持上下文）
- 快捷问题（7个预设场景）
- 语音输入转文字（浏览器 Web Speech API）
- 服务推荐（基于意图识别）
- 历史记录管理

## 十一、通知服务

### 11.1 当前实现（MVP）

站内消息系统，通过 `messages` 表/集合存储：

```javascript
addMessage(db, toRole, title, content, meta)
// toRole: "user" | "family" | "guide" | "merchant" | "admin"
// meta: { scenario, priority, channels, relatedType, relatedId }
```

### 11.2 通知触发场景

| 场景 | 接收方 | 优先级 |
|------|--------|--------|
| 订单创建 | admin | 普通 |
| SOS触发 | admin + family | 紧急 |
| 派单完成 | guide/merchant | 普通 |
| 任务完成 | user | 普通 |
| 设备异常 | user + family + admin | 高 |
| 快速求助 | admin + guide | 紧急 |

### 11.3 预留通道

- 站内消息 ✅（已实现）
- 短信通知（接口预留）
- 电话通知（接口预留）
- 微信模板消息（二期对接）
- WebSocket 实时推送（二期实现）
