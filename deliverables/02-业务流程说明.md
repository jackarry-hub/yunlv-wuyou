# 云旅无忧 AI 智慧旅居平台 — 业务流程说明

> 版本：v1.0.0 | 更新日期：2026-06-14

## 一、总体业务流程（七步闭环）

```
用户发起需求 → 智能分析匹配 → 任务分配派单 → 服务执行跟踪 → 反馈上报确认 → 结果通知评价 → 数据沉淀闭环
```

### 1. 需求发起
- 用户在用户端提交服务订单（陪伴就医、导游游览、护工护理、接送出行等）
- AI 管家自动解析意图并推荐匹配服务
- 触发端：用户端 → API：`POST /api/orders`

### 2. 智能分析匹配
- 系统根据服务类型、位置、时间、向导/商户资质自动匹配候选
- AI 管家辅助分析需求，生成推荐方案
- 触发端：后台自动 → API：`GET /api/admin/dispatch/candidates`

### 3. 任务分配派单
- 管理后台运营人员审核并派单
- 支持自动派单规则：距离优先 + 评分优先 + 服务类型匹配
- 触发端：管理后台 → API：`POST /api/tasks/dispatch`

### 4. 服务执行跟踪
- 向导/商户接单并开始服务
- 实时状态更新：待接单 → 已接单 → 服务中
- 支持路线导航、异常上报
- 触发端：向导端/商户端 → API：`POST /api/tasks/{id}/accept|start`

### 5. 反馈上报确认
- 向导/商户完成服务后上报
- 用户确认服务完成
- 触发端：向导端/商户端 → API：`POST /api/tasks/{id}/complete`

### 6. 结果通知评价
- 系统自动通知用户服务完成
- 用户进行评价
- 触发端：系统自动 → API：`POST /api/reviews`

### 7. 数据沉淀闭环
- 所有操作写入审计日志
- 订单/评价/健康/消息数据沉淀
- 后台数据大屏实时展示
- API：`GET /api/admin/data-loop`

## 二、订单状态流转

```
待派单 → 已派单 → 已接单 → 服务中 → 已完成
                                    ↘ 已取消
```

| 状态 | 触发方 | 说明 |
|------|--------|------|
| 待派单 | 用户下单 | 用户提交订单后初始状态 |
| 已派单 | 后台派单 | 管理后台将订单分配给向导/商户 |
| 已接单 | 向导/商户 | 执行方确认接单 |
| 服务中 | 向导/商户 | 执行方开始服务 |
| 已完成 | 向导/商户 | 服务完成上报 |
| 已取消 | 用户/后台 | 取消订单 |

## 三、紧急求助流程

```
用户触发SOS → 系统生成告警 → 通知后台+紧急联系人 → 后台处理 → 结果反馈
```

1. 用户一键 SOS：`POST /api/alerts/sos`
2. 系统自动通知后台和紧急联系人
3. 后台客服处理异常：`POST /api/alerts/{id}/handle`
4. 处理结果同步回用户端消息

## 四、商户服务流程

```
商户入驻审核 → 服务发布上架 → 接收派单/预约 → 报价确认 → 服务执行 → 完成上报 → 评价结算
```

| 步骤 | 操作方 | API |
|------|--------|-----|
| 入驻审核 | 管理后台 | `POST /api/admin/merchants/{id}/audit` |
| 服务发布 | 商户端 | `POST /api/merchant/services` |
| 报价确认 | 商户端 | `POST /api/merchant/orders/{id}/quote` |
| 确认预约 | 商户端 | `POST /api/merchant/orders/{id}/confirm` |
| 开始服务 | 商户端 | `POST /api/merchant/orders/{id}/start` |
| 完成上报 | 商户端 | `POST /api/merchant/orders/{id}/complete` |

## 五、向导服务流程

```
向导认证 → 上线接单 → 接单大厅待接 → 接单确认 → 路线导航 → 服务中 → 完成上报
```

| 步骤 | 操作方 | API |
|------|--------|-----|
| 向导认证 | 管理后台 | `POST /api/admin/guides/{id}/audit` |
| 上线接单 | 向导端 | `POST /api/guide/online` |
| 抢单/接单 | 向导端 | `POST /api/guide/tasks/claim-next` |
| 接受任务 | 向导端 | `POST /api/tasks/{id}/accept` |
| 开始服务 | 向导端 | `POST /api/tasks/{id}/start` |
| 完成上报 | 向导端 | `POST /api/tasks/{id}/complete` |
| 异常上报 | 向导端 | `POST /api/guide/exception` |

## 六、跨端协同通知规则

| 场景 | 触发端 | 接收端 | 通知内容 |
|------|--------|--------|----------|
| 用户下单 | 用户端 | 管理后台 | 新订单待派单 |
| 后台派单 | 管理后台 | 向导/商户端 | 新任务待接单 |
| 执行方接单 | 向导/商户端 | 用户端 | 已接单服务中 |
| 服务完成 | 向导/商户端 | 用户端+后台 | 服务已完成请评价 |
| SOS 求助 | 用户端 | 后台+紧急联系人 | 紧急求助请处理 |
| 设备异常 | 系统 | 后台+用户端 | 设备离线/异常 |
| 商户报价 | 商户端 | 用户端 | 商户已报价待确认 |
| 异常上报 | 向导/商户端 | 管理后台 | 服务异常待处理 |

## 七、AI 管家服务流程

```
用户提问/语音 → 意图识别 → 知识库检索 → 生成回答 → 服务推荐 → 一键下单
```

1. 用户输入文字或语音：`POST /api/ai/chat` / `POST /api/ai/voice/transcribe`
2. 系统识别意图（活动推荐、服务咨询、健康建议等）
3. 基于知识库和用户档案生成个性化回答
4. 推荐匹配服务并支持一键下单
5. 对话记录沉淀：`GET /api/ai/history`
